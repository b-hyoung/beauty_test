# BTest 기획서 (필수 요약)

## 1. 목적
- 사용자 얼굴 사진 1장으로 피부 상태를 빠르게 분석하고, 이해 가능한 결과와 개선 가이드를 제공한다.

## 2. 대상 사용자
- 피부 상태를 간단히 점검하고 싶은 일반 사용자
- 병원 방문 전, 셀프 체크가 필요한 사용자

## 3. 핵심 사용자 흐름
1. 랜딩(`/`) 진입
2. 분석 페이지(`/views`) 이동
3. 사진 가이드라인 확인 후 얼굴 사진 업로드
4. 서버 분석 요청(`/api/skin-precheck` -> `/api/skin-analyze`)
5. 결과 점수/요약 확인
6. 개선 시뮬레이션 및 추천 루틴 확인

## 4. 기능 범위
### 포함
- 랜딩 페이지
- 사진 업로드(드래그 앤 드롭/파일 선택)
- 업로드 전 가이드라인 표시
- 얼굴 사진 사전검증(precheck)
- 피부 분석 점수/요약 결과 표시
- 개선 시뮬레이션 탭 UI 및 루틴 추천

- 회원가입/로그인
- 분석 이력 저장
- 결제/구독

## 5. API 구성
- `POST /api/skin-precheck`
  - 입력: 이미지 파일
  - 역할: 실사 정면 얼굴 여부 사전 검증
- `POST /api/skin-analyze`
  - 입력: 이미지 파일
  - 역할: 피부 지표 분석 및 UI용 결과 반환

## 6. 라우팅 정책
- 기본 진입: `/`
- 분석 화면: `/views`
- 호환 리다이렉트:
  - `/btest` -> `/`
  - `/btest/views` -> `/views`

## 7. 품질 기준(완료 정의)
- 업로드부터 결과 표시까지 사용자 흐름이 중단 없이 동작
- 필수 에러(사진 미선택, precheck 실패, analyze 실패) 안내 문구 노출
- Cloudflare 도메인 개발 접속 시 CORS/Dev Origin 경고 없이 동작

## 8. 데이터 저장 정책
- 서버 DB 저장: 사용하지 않음
- 서버 파일 저장: 사용하지 않음
- 분석 결과는 요청-응답 처리 후 서버에 영구 저장하지 않음
